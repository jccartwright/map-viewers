      Abstract: 
          These data were generated to provide insight into traffic patterns on a macro scale so that they could be 
          analyzed across the coastal waters of the Continental United States. For this dataset, a transit is counted 
          for each unique vessel intersecting a 1 kilometer square grid cell each day. This data represents the total 
          number of vessel transits from October 2009 to October 2010. There were some grid cells which could not be 
          processed, but we do not believe this interferes with the integrity of this dataset. Please note multiple 
          connection errors occurred during the time frame of this study. Where possible, data gaps were filled by 
          requests to the Coast Guard or other organizations that received the same data feed. However, due to 
          resource constraints, uninterrupted coverage was not obtained. Overall data outages were minimal, on the 
          order of less than a day per month. Because the outages were random and affected all areas uniformly, they 
          do not have a significant effect on the integrity of the data. Also, as stated on the United States Coast 
          Guard (USCG) Nationwide Automatic Identification System (NAIS) website: AIS data is not representative of 
          all vessel traffic, and USCG NAIS receivers do not fully cover the entire extent of this study area. Please 
          take time to understand both of these limitations. Refer to the NAIS website for more details: 
          http://www.navcen.uscg.gov/?pageName=NAISmain.
      Purpose: 
          To support coastal and ocean planning and other activities pursuant to the Energy Policy Act, Coastal Zone 
          Management Act, Magnuson-Stevens Fishery Conservation and Management Act, National Environmental Policy Act, 
          Rivers and Harbors Act and the Submerged Lands Act.

              Description: 
                  Futher data development processing steps were followed to obtain this feature class: Acquired original 
                  source data and maintained a copy on the D-tier (data01_d), 
                  \\csc-s-san1b\marinsp\MultipurposeMarineCadastre\original\2012\OCS AIS GRIDS. Created a geodatabase 
                  in a personal workspace and imported the original shapefiles into this file geodatabase format. 
                  Added a field in each of the feature classes on which to base classification ranges and deleted all 
                  values of "0" in each of the data sets. Using the field calculator, populated the new classification 
                  field values based on selecting the values of 'TRNSTS_TTL' field; this was performed on each UTM 
                  zone feature class. Then performed a "dissolve" on each of the individual UTM zones datasets based 
                  on the classification field. Created a new feature class with NAD 83 projection, added a field for 
                  the classification code, and loaded the data from the classified dissolved dataset into the new NAD 
                  83 dataset. Validated the feature class geometry using "Data Management > Features > Repair 
                  Geometry". Added a field in each of the three new data sets called 'vesselCountRange, populated 
                  this new field using field calculator to display the actual range of values within that 
                  classification code. When necessary, opened ArcMap and in a new MXD, clicked File > Map Document 
                  Properties and checked the box Store relative path names to data sources. Added in the Oceans 
                  Basemap in the working MXD to work up the color scheme for the data. Selected the fields to make 
                  visible (chose to hide Classification Code and show Vessel Count Range). Symbolized the layer by 
                  Classification Code field to represent the Vessel Count Ranges. Developed a layer file containing 
                  symbology from each feature class. Included Layer Name, Description, Credits, and Scale Ranges (if 
                  applicable) within the layer file and its source. Arranged quality process review by peers. Reviewed 
                  quality process feedback, determined applicability, and made adjustments to data to meet process 
                  specifications. Placed a copy of the feature class/es within ArcSDE. Projected the symbolized layer 
                  into Web Mercator and placed within the viewer's MXD. Worked with the MXD steward to determine where 
                  and how the layer fit within the MXD. Acquired, created and updated metadata from providers (as 
                  needed). Planned to annually evaluate source data to determine requirement for replacement.
