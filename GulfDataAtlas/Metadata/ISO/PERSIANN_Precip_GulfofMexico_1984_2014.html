      Abstract: 
          The Precipitation Estimation from Remotely Sensed Information using an Artificial Neural Network-Climate Data 
          Record (PERSIANN-CDR) is a new, retrospective satellite-based precipitation dataset, constructed as a 
          climate data record for hydrological and climate studies. The PERSIANN-CDR is available from 1983-present 
          making the dataset the longest satellite based precipitation data record available. The precipitation maps 
          are available at daily temporal resolution for the latitude band 60°S–60°N at 0.25 degrees. The maps shown 
          here represent 30-year annual and seasonal median and interquartile range (IQR) of the PERSIANN-CDR dataset 
          from 1984 – 2014. In the median precipitation maps, the mid-point value (or 50th percentile) for each pixel 
          in is computed and plotted for the study area. The range of the data about the median is represented by the 
          interquartile range (IQR), and shows the variability of the dataset. The interquartile range is the 
          difference between the 3rd quartile and the 1st quartile of the dataset, also computed at each pixel for 
          daily values. The difference between the quartiles is plotted for the study area.For these maps, winter = 
          December – February, spring = March – May, summer = June – August, fall = September – November
      Purpose: 
          These estimated precipitation climatology maps were generated for display in the NOAA Gulf of Mexico Data 
          Atlas to increase awareness, and facilitate data discovery and access of the PERSIANN CDR data product 
          developed by the National Centers for Environmental Information's (NCEI) Climate Weather Center (CWC).

          Statement: 
              This dataset consists of raster map layers showing estimated precipitation values, and contour map layers 
              generated from the raster data.
          Process step: 
            Process Step: 
              Description: 
                  Daily preciptation estimate netCDF files for 1983-2015 were extracted from the PERSIANN THREDDS 
                  server. Daily files from December 1983-November 2014 were read into an R project to create a 
                  pixel-wise sum for each year and each season across the period of record. Summed data were saved as 
                  netCDF files. R was also used to compute annual and seasonal medians and interquartile ranges of the 
                  summed data. Each sum, median, and interquartile range file was examined visually and exported as 
                  geotiffs. The geotiffs were imported in to ArcMap desktop and a min-max stretch across a color ramp 
                  was applied to the the annual and seasonal median images, and the annual and seasonal interquartile 
                  range images using total range of the annual and seasonal data for each.
              Date and time: 
                  2017-08-25T00:00:00
          Process step: 
            Process Step: 
              Description: 
                  Contour files for each geotiff were created in ArcMap desktop using the 3D Analyst/Raster 
                  Surface/Contour tool. The median annual data were run at 200 mm contour intervals, the seasonal data 
                  were run at 100 mm contour intervals, and all the IQR data were run at 50 mm contour intervals. The 
                  resulting contour layers were manually cleaned up in an edit session, then smoothed using the 
                  Cartography/Generalization/Smooth Line tool to a tolerance of 2 decimal degrees. The lines were then 
                  symbolized and labeled.
